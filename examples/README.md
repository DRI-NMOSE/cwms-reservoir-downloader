# Examples

End-to-end scripts that download data via the CWMS Data API and render a
multi-panel plot per reservoir. Each script picks one reservoir in one USACE
district, downloads the last year of three time series, and saves a PNG under
[figures/](figures/).

## Setup

```bash
pip install -e ".[examples]"   # installs matplotlib
```

## Generated CSVs

Example 04 also writes the full pool-elevation record for every NM reservoir
to [data/nm_elevations/](data/nm_elevations/) — one CSV per reservoir, named
`<location>__<ts_id>.csv`, with the raw `date-time`, `value`, and quality
columns returned by the CWMS Data API. The directory is git-ignored (see the
project-wide `data/` rule) since it can be regenerated by re-running the
script.

The 12 NM reservoirs covered, along with the earliest timestamp present in
each downloaded series. Example 04 prefers the long-history `DCP-rev`
district SCADA feed when it exists and falls back to the short-history `MRS`
series otherwise:

| Reservoir   | Time series ID                              | Record starts |
| ----------- | ------------------------------------------- | ------------- |
| Abiquiu     | `Abiquiu.Elev.Inst.15Minutes.0.DCP-rev`     | 2004-02-25    |
| Brantley    | `Brantley.Elev.Inst.15Minutes.0.DCP-rev`    | 1999-03-01    |
| Cochiti     | `Cochiti.Elev.Inst.15Minutes.0.DCP-rev`     | 2004-02-25    |
| Conchas     | `Conchas.Elev.Inst.15Minutes.0.DCP-rev`     | 2004-02-11    |
| Diamond A   | `Diamond A.Elev.Inst.~1Day.0.MRS`           | 2026-02-02    |
| Galisteo    | `Galisteo.Elev.Inst.~1Day.0.MRS`            | 2026-02-01    |
| Jemez       | `Jemez.Elev.Inst.~1Day.0.MRS`               | 2026-02-01    |
| Navajo      | `Navajo.Elev.Inst.1Hour.0.DCP-rev`          | 2010-09-21    |
| Rocky       | `Rocky.Elev.Inst.~1Day.0.MRS`               | 2026-02-02    |

Reservoirs without a `DCP-rev` feed (Diamond A, Galisteo, Jemez, Rocky, Two
Rivers) fall back to the `MRS` series, which on the live CDA endpoint only
retains roughly the last few months of data — so those rows in the combined
plot will appear as a short recent segment. Re-running the script refreshes
both the CSVs and the combined figure to today's date.

## Adapting to another reservoir

1. Copy one of the three scripts.
2. Change `state`, `office`, `location`, and the `series` tuple in the
   `ReservoirExample` block. Each `SeriesSpec` is a real CWMS time series ID
   (the dotted `Loc.Param.Type.Interval.Duration.Version` string).
3. Find the available IDs for your reservoir by running:

   ```bash
   cwms-reservoir-downloader --state <STATE> --list-only
   # then inspect data/<STATE>/timeseries_catalog.csv after a real run
   ```

The shared helper [_plot_helpers.py](_plot_helpers.py) handles session init,
fetching, and figure layout — the per-state scripts only declare *what* to
plot.
